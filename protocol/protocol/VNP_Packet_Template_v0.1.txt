---------------------------------------------------------------------
Usage
---------------------------------------------------------------------
This packet serves as a template for creating new VNP-compliant messages.
Each message begins with the header block above, placed at the start of
a .tex or .pdf source file. The header defines routing, verification,
and identification data for inter-node communication.

---------------------------------------------------------------------
Field Overview
---------------------------------------------------------------------
Field        | Description
--------------|------------------------------------------------------
to            | Destination node (recipient)
from          | Originating node (sender)
role          | Assigned node function (e.g., Maintainer, Reviewer)
ref_project   | Project or repository reference identifier
task_id       | Unique task or transaction number (YYYY-MM-DD-###)
checksum      | SHA-256 hash of document body (integrity check)
intent        | Brief description of task or transmission purpose

---------------------------------------------------------------------
Instructions
---------------------------------------------------------------------
1. Replace all header values with message-specific details.
2. Maintain the same structure and spacing for compatibility.
3. Compute the SHA-256 checksum on the completed body (excluding header).
4. Embed the header at the start of the .tex source file or PDF metadata.
5. Confirm routing and intent before transmission.
